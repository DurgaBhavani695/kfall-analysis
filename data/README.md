# Data Directory

This directory holds the KFall dataset files required to run the pipeline.

## Expected structure

```
data/
├── raw/
│   ├── k_fall.csv               ← activity catalogue
│   ├── sensor_data/
│   │   └── <SubjectId>/
│   │       └── <SubjectId>T<TaskCode>R0<TrialId>.csv
│   └── label_data/
│       └── <SubjectId>_labels.xlsx
└── processed/
    └── data.csv                 ← auto-generated by train.py
```

## Obtaining the KFall dataset

The KFall dataset is publicly available from the original paper:

> Kim, J. et al. (2021). *KFall: A Multi-Modal Dataset for Human Fall Detection Using Wearable Inertial Sensors*.  
> [https://www.mdpi.com/1424-8220/21/16/5388](https://www.mdpi.com/1424-8220/21/16/5388)

1. Download `sensor_data.zip`, `label_data.zip` and `k_fall.csv` from the
   dataset repository / supplementary materials.
2. Unzip `sensor_data.zip` → `data/raw/sensor_data/`
3. Unzip `label_data.zip` → `data/raw/label_data/`
4. Place `k_fall.csv` at `data/raw/k_fall.csv`

> **Note:** Raw data files are excluded from version control via `.gitignore`
> to avoid committing large binary/CSV files to the repository.

## Processed data

`data/processed/data.csv` is automatically generated the first time you run:

```bash
python scripts/train.py
```

You can also regenerate it independently:

```bash
python scripts/train.py --skip-load   # skip loading if processed CSV exists
```
